SQLite3 Actions Plugin

 Online Help

Description

SQLite3 is an action plugin that makes use of the SQLite Embeddable SQL Database Engine to add high-performance database functionality to your application.

Note: The SQLite database engine is not a client used to connect to a big database server. It is the server. The SQLite database engine reads and writes directly to and from database files on disk.

Highlights of the SQLite Database Engine:

  • Implements most of SQL92. (Features not supported)
  • Each complete database (with multiple tables and indices) is stored in a single disk file.
  • Atomic commit and rollback protect data integrity.
  • Database files can be freely shared between machines with different byte orders.
  • Supports databases up to 2 terabytes (2^41 bytes) in size.
  • Two times faster than PostgreSQL and MySQL for many common operations.
  • Self-contained: no external dependencies!

For more information on the SQLite database engine, see the SQLite website at www.sqlite.org.

Quick Start

Working with a SQLite database consists of three basic steps: opening a database, querying the database, and then closing the database once you're done with it.

Opening a database

The first step to working with a SQLite database is to open an existing database file (or create a new database file) using SQLite3.Open():

db = SQLite3.Open("sesame.db");

Note that SQLite3.Open returns a database handle, which is needed for most other SQLite actions. Each handle uniquely identifies one open database, so you can use multiple handles to interact with several databases at the same time.

Querying the database

While you have a database open, you can use the SQLite Query Language to interact with the database.

There are three basic methods you can use to perform a SQLite query on the database:

  • Use the SQLite3.Query() action. This is the best method to use when executing query statements that don't return any results. For example:

    SQLite3.Query(db, "create table Users(userid integer primary key, LastName text, FirstName text, Age integer)");

    If you need to process the query results, you can provide an optional callback function and it will be called once for every row in the result set. For example:

    function myfunc(db, nRow, nColumns, tbRowDataAssoc, tbRowDataArray, tbColumnNamesArray, tbColumnTypesArray, tbColumnTypesAssoc)
    -- do something here
end

sql = "select * from users";
SQLite3.Query(db, sql, myfunc);

  • Use the SQLite3.QueryToTable() action, which returns the results of the query in a table. For example:

    sql = "select * from users";
tbResults = SQLite3.QueryToTable(db, sql);

-- get the value in FirstName in row 1 of the result set
first_name = tbResults.Data[1]["FirstName"];

  • Use the SQLite3.Compile() action to compile the query into a "virtual machine" so you can step through query one row at a time. For example, your would first compile a query like this:

    SQLite3.Compile(db, "select * from users");

    Once you have the query compiled, you can repeatedly call SQLite3.Step() to process it:

    repeat
    SQLite3.Step(db);
    err = Application.GetLastError();
until  err ~= SQLite3.ROW

    At each step, you can retrieve the current row of data by using one of the "GetRow..." actions:

    tbRow = SQLite3.GetRowDataAssoc(db);
first_name = tbRow["FirstName"];

    Once you're done with a compiled query, you must call SQLite3.Finalize() to finalize it: 

    SQLite3.Finalize(db);

Choosing Between the Three Query Methods


SQLite3.QueryToTable() is the most straightforward method...it just performs the entire query immediately and returns the results as a Lua table. This is the easiest one to work with, however it doesn't allow you to abort the query at all, and (at least theoretically) needs more memory than the other two actions, since the entire results of the query are copied into a Lua table.

SQLite3.Query() and SQLite3.Compile()/Step()/Finalize() are essentially two different ways of doing the same thing: performing a query that gives you the results one row at a time and that you can stop after any row. They both have the same effect, but allow you to achieve that effect in different ways.

SQLite3.Query() is the best choice for simple queries where you aren't trying to get any results back, but are just performing some action on the database (e.g. creating a table). The reason it's the best choice is simply because it lets you get the job done with 1 action instead of 3.

However, SQLite3.Query doesn't return any data unless you use a callback function.

The other three actions (Compile, Step and Finalize) are provided as an alternative so you don't have to use a callback function. Instead, you "start" a query with SQLite3.Compile(), and then "step" through the results with SQLite3.Step(). SQLite3.Step() actually perfoms another "step" of the query and puts the results into memory where the 5 result-getting actions (the "GetRow..." actions) can access them. When you're done getting results for this query, you "stop" it with SQLite3.Finalize().

The decision of whether to use SQLite3.Query() or SQLite3.Compile()/Step()/Finalize() is really just personal preference.

Closing the database

When you're done with a database, you should call SQLite3.Close() to close it.

SQLite3.Close(db);

Database conversion

Converting an existing database from SQLite2 to SQLite3


I you have an existing database created with SQLite 2.x and you desire to use it with the new SQLite3 engine, then you need to convert it first. This can be done easily with a couple of steps, as shown below.

In the "Plugins\Actions\SQLite3" sub folder of your AutoPlay Media Studio installation folder (or "Includes\Plugins\SQLite3" if you are using Setup Factory 9), you will find two programs, named sqlite.exe and sqlite3.exe. These are the command line tools to work with SQLite databases directly.

First, convert your SQLite2 database to a SQL text file, where database.db is the name of the file you wish to convert: 

sqlite.exe database.db .dump > database.sql

Now, use the SQL dump file and create a new SQLite3 database from it, using this command line syntax:

sqlite3.exe newdatabase.db < database.sql

This will create a new database with the same contents as your previous version. You may notice that the new file is smaller - this is one of the many benefits of using the newer version of SQLite.

SQLite3.Close (database dbHandle);

Description:

Closes a SQLite database.

Note: This releases the database handle and reclaims any memory that the database was using.

(database) The handle to the database that you want to close.

Nothing.

number SQLite3.Compile (database dbHandle, string Query);

Description:

Compiles a query so it can be stepped through one row at a time using SQLite3.Step.

dbHandle:

(database) The handle to the database that you want to operate on.

Query:

(string) The SQLite query string that you want to perform. See the SQLite Query Language for more information.

Returns:

(number) The length of the query string that was processed.

Example:

-- Compile a query in the database

if nLen ~= len(sql) then
    Debug.Print("Possible error: length of compiled query doesn't match original query length");
end

Description:

if err == SQLite3.ERROR then
    strErrMsg = SQLite3.GetLastErrorString();

end

number SQLite3.GetLastInsertRowID (database dbHandle);

Description:

Returns the row ID of the most recent insert in the database.

Note: This action is similar to the mysql_insert_id() function in MySQL.

dbHandle:

(database) The handle to the database that you want to operate on.

Returns:

(number) The ID of the row that was modified by the last "insert" operation.

Note: Each entry in an SQLite table has a unique integer key. The key is the value of the INTEGER PRIMARY KEY column if there is such a column, otherwise the key is generated at random.

Example:

-- which row was modified?
nRID = SQLite3.GetLastInsertRowID(db);

database SQLite3.Open (string Filename);

Description:

Opens a SQLite3 database. If the database does not exist, it is created. This action will not open database files created in previous versions of SQLite, like 2.x.

Filename:

(string) The full path and filename of the database file that you want to open. Use ":memory:" to open a temporary database that only exists in the computer's memory.

Note: The special ":memory:" filename is primarily useful for temporary processing, as the in-memory database will be destroyed when the database is closed.

Returns:

(database) A handle to the database that you can use in other SQLite actions.

Note that you can open multiple databases at the same time. Each database connection is identified by a unique database handle.

Example 1:

-- open a database file in the user's temp folder
handle = SQLite3.Open(_TempFolder .. "\\MyDatabase.db");

Example 2:

-- open a database file in memory and create a 3-column table in it
db = SQLite3.Open(":memory:");
SQLite3.Query(db,"CREATE TABLE Users(userid INTEGER PRIMARY KEY, LastName TEXT, FirstName TEXT, Age INTEGER)");

SQLite3.Query (database dbHandle, string Query, string CallbackFunction, boolean bConvert);

Description:

Performs a query on an SQLite database.

dbHandle:

(database) The handle to the database that you want to operate on.

Query:

(string) The SQLite query string that you want to perform. See the SQLite Query Language for more information.

CallbackFunction:

(string) The name of a function that will be called for every row returned from the query. (You can use this callback function to collect the results returned from the query.)

bConvert:

(boolean) Optional, default value is "false". Specifies if the data should be converted from UTF-8 to MBCS. See sample here.

Note: The CallbackFunction is optional...if you don't need to process the results, you may set this parameter to nil (or omit it completely).

The callback function must be able to receive the following parameters:

dbHandle:

(database) A handle to the database that is being operated on.

nRow:

(number) The current row in the query result. (This number gets incremented each time the callback is called.)

nColumns:

(number) The number of columns in the query result.

tbRowDataAssoc:

(table) A table containing the row data indexed by column name.

If the EMPTY_RESULT_CALLBACKS pragma is on, and the query didn't return any results, tbRowDataAssoc will be nil.

tbRowDataArray:

(table) A table containing the row data indexed numerically.

If the EMPTY_RESULT_CALLBACKS pragma is on, and the query didn't return any results, tbRowDataArray will be nil.

tbColumnNamesArray:

(table) A table containing the column names indexed numerically.

tbColumnTypesArray:

(table) A table containing the column types indexed numerically.

If the SHOW_DATATYPES pragma is off, tbColumnTypesArray will be nil.

tbColumnTypesAssoc:

(table) A table containing the column types indexed by column name.

If the SHOW_DATATYPES pragma is off, tbColumnTypesAssoc will be nil.

Note: The SHOW_DATATYPES pragma is off by default.

The callback function should return a boolean value (true or false) indicating whether the query should continue:

true - Continue performing the query.
false - Abort the query.

Returns:

Nothing.

Example 1:

-- open a temporary database in memory
db = SQLite3.Open(":memory:");

-- create a table with 3 columns
SQLite3.Query(db,"create table foo(a integer primary key, b text, c integer)");");

Example 2:

--[[ cbPrintRow function.
     Use this to print a row of data to the debug window (assumes the debug window is already open). ]]
function cbPrintRow(db, nRow, nColumns, tbRowDataAssoc, tbRowDataArray, tbColumnNamesArray, tbColumnTypesArray, tbColumnTypesAssoc)

    Debug.Print("**  ROW " .. nRow .. " IN RESULT SET HAS " .. nColumns .. " COLUMNS **\r\n");

    --[[ note: always make sure tbRowDataAssoc, tbRowDataArray,
               tbColumnTypesArray, and tbColumnTypesAssoc are NOT nil
               before working with them. An easy way to do this is to
               test those variables to see if they contain something
               that is considered "true." Only the values nil and false
               are considered "false"...anything else (including a table)
               is considered true. ]]

    if tbRowDataAssoc then

        Debug.Print("\r\nRow ".. nRow .." data (associative style):\r\n");

        for strColumnName, strData in pairs(tbRowDataAssoc) do
            nCol = 1;
            Debug.Print(strColumnName .. " = " .. strData);
    		if nCol ~= nColumns then
    			Debug.Print(", ");
    		end
        end
        Debug.Print("\r\n");
    end

    if tbRowDataArray then

        Debug.Print("\r\nRow ".. nRow .." data (array style):\r\n");

        for nCol = 1,nColumns do
            Debug.Print(tbColumnNamesArray[nCol] .. " = " .. tbRowDataArray[nCol] );
    		if nCol ~= nColumns then
    			Debug.Print(", ");
    		end
        end
        Debug.Print("\r\n");
    end

    if tbColumnTypesAssoc then

        Debug.Print("\r\nColumn types (associative style):\r\n\r\n");

        for strColumnName, strColumnType in pairs(tbColumnTypesAssoc) do
            Debug.Print(strColumnName .. " type: " .. strColumnType .. "\r\n");
        end
    end

    if tbColumnTypesArray then
        Debug.Print("\r\nColumn types (array style):\r\n\r\n");

        for i, strColumnType in pairs(tbColumnTypesArray) do
            Debug.Print ("Column " .. i .. " type: " .. strColumnType .. "\r\n");
        end
    end

    Debug.Print("\r\n");

    -- continue with the query
    return true;
end


-- create a database in memory
db = SQLite3.Open(":memory:");
err = Application.GetLastError();
if err == SQLite3.OK then
    -- create a table and fill it with data
    SQLite3.Query(db,"create table Users(userid integer primary key, LastName text, FirstName text, Age integer)");
    SQLite3.Query(db,"insert into Users values(1,'Sellers','Ted',48)");
    SQLite3.Query(db,"insert into Users values(2,'Blow','Joe',64)");
    SQLite3.Query(db,"insert into Users values(NULL,'Dandy','Jim',31)");
    SQLite3.Query(db,"insert into Users values(NULL,'Osborne','Super Dave',47)");
    SQLite3.Query(db,"insert into Users values(NULL,'Hakimoto','Fuji',52)");

    --[[ note: to keep this example from getting too complicated, we'll assume that
               the table can be created and filled with no errors. In practice,
               you should check for errors after each call to SQLite3.Query. ]]

    -- open the debug window
    Debug.ShowWindow(true);

	-- turn on the SHOW_DATATYPES option in this database
	-- so the callback function will display the column types
	SQLite3.Query(db,"PRAGMA SHOW_DATATYPES=ON;")

    -- Perform a query and use our callback function to print the results to the debug window
    SQLite3.Query(db,"SELECT * FROM Users", "cbPrintRow");
    err = Application.GetLastError();
    if err ~= SQLite3.OK then
        Debug.Print( "Error: " .. SQLite3.GetErrorString(err) );
    end

    -- close the database
    SQLite3.Close(db);
else
    -- error trying to open the database
    Debug.Print( "Error: " .. SQLite3.GetErrorString(err) );
end

SQLite3.QueryToTable (database dbHandle, string Query, boolean bConvert);

Description:

Performs a query on an SQLite database and returns a table containing the results.

dbHandle:

(database) The handle to the database that you want to operate on.

Query:

(string) The SQLite query string that you want to perform. See the SQLite Query Language for more information.

bConvert:

(boolean) Optional, default value is "false". Specifies if the data should be converted from UTF-8 to MBCS. See sample here.

Returns:

(table) A table containing the query results. If an error occurs, this action returns nil.

If successful, the following information will be stored in the table:

Rows:
(number) The number of rows in the query result.

Columns:
(number) The number of columns in the query result.

ColumnNames:
(table) A numerically indexed table containing the names of the columns in the query result.

Use .ColumnNames[<column_number>] to get the name of a column:
column_three = tbResult.ColumnNames[3];

Data:
(table) A table containing the data returned by the query. The data is organized into sub-tables, one for each row in the query result. Each sub-table contains the data from one row, indexed by column name.

Use .Data[<row_number>] to access a row of data:
tbRow2 = tbResult.Data[2];

Then use [<column_name>] to access a cell value:
strLastName = tbRow2["LastName"];

Or, use .Data[<row_number>][<column_name>] to access a cell value directly:
strLastName = tbResult.Data[2]["LastName"];

Example 1:

tbResult = SQLite3.QueryToTable(mydb, "select * from players,teams where teamid=7;");

-- get the name of player 7:
tbRow2 = tbResult.Data[2];
player_name = tbRow2["FirstName"] .. " " .. tbRow2["LastName"];

Example 2:

tbResult = SQLite3.QueryToTable(mydb, "select * from players,teams where teamid=7;");

-- get the name of player 7:
player_name = tbResult.Data[2]["FirstName"] .. " " .. tbResult.Data[2]["LastName"];

Example 3:

-- open the debug window
Debug.ShowWindow(true);

-- create a database in memory
dbname = ":memory:";
db = SQLite3.Open(dbname);
err = Application.GetLastError();
if err == SQLite3.OK then

    Debug.Print("Opening database: \"" .. dbname .. "\"\r\n\r\n");

    -- create a table and fill it with data
    SQLite3.Query(db,"create table Users(userid integer primary key, LastName text, FirstName text, Age integer)");
    SQLite3.Query(db,"insert into Users values(1,'Sellers','Ted',48)");
    SQLite3.Query(db,"insert into Users values(2,'Blow','Joe',64)");
    SQLite3.Query(db,"insert into Users values(NULL,'Dandy','Jim',31)");
    SQLite3.Query(db,"insert into Users values(NULL,'Osborne','Super Dave',47)");
    SQLite3.Query(db,"insert into Users values(NULL,'Hakimoto','Fuji',52)");

    --[[ note: to keep this example from getting too complicated, we'll assume that
               the table can be created and filled with no errors. In practice,
               you should check for errors after each call to SQLite3.Query. ]]

    -- perform a query and store all of the results in a table called tbUsers
    tbUsers = SQLite3.QueryToTable(db,"SELECT * FROM Users");
    err = Application.GetLastError();
    if err ~= SQLite3.OK then
        Debug.Print( "Error: " .. SQLite3.GetErrorString(err) );
    end

    -- display the contents of the table

    Debug.Print("Results returned from QueryToTable:\r\n\r\n");

    Debug.Print(tbUsers.Columns .." Columns\r\n");
    Debug.Print(tbUsers.Rows    .." Rows\r\n\r\n");

    for n = 1,tbUsers.Columns do
    	Debug.Print("Name of column "..n.." is: "..tbUsers.ColumnNames[n].."\r\n");
    end

    Debug.Print("\r\n\r\n");

    for nRow = 1,tbUsers.Rows do
        Debug.Print("Row "..nRow..":\r\n\r\n");
        Debug.Print("   User ID: "..tbUsers.Data[nRow]["userid"].."\r\n");
        Debug.Print("   First name: "..tbUsers.Data[nRow]["FirstName"].."\r\n");
        Debug.Print("   Last name: "..tbUsers.Data[nRow]["LastName"].."\r\n");
        Debug.Print("   Age: "..tbUsers.Data[nRow]["Age"].."\r\n\r\n");
    end

    for nRow = 1,tbUsers.Rows do
        Debug.Print("Row "..nRow..": ");
    	for nCol = 1, tbUsers.Columns do
    		strColumnName = tbUsers.ColumnNames[nCol];
    		Debug.Print(strColumnName .."="..tbUsers.Data[nRow][strColumnName]);
    		if nCol ~= tbUsers.Columns then
    			Debug.Print(", ");
    		end
    	end
    	Debug.Print("\r\n");
    end

    -- close the database
    Debug.Print("\r\nClosing database: \"" .. dbname .. "\"\r\n\r\n");
    SQLite3.Close(db)

else
    -- error trying to open the database
    Debug.Print( "Error: " .. SQLite3.GetErrorString(err) );
end

SQLite3.SetBusyTimeout (database dbHandle, number Timeout);

Description:

SQLite3.Step (database dbHandle);

Description:

Advances a compiled query to the next row.

After a query has been compiled using SQLite3.Compile, it is actually performed by one or more calls to SQLite3.Step. Each invocation of SQLite3.Step, except the last one, returns a single row of the result. To access these results, use one of the following actions:

GetRowDataArray
GetRowDataAssoc
GetRowColumnNamesArray
GetRowColumnTypesArray
GetRowColumnTypesAssoc

You can use Application.GetLastError() immediately after calling SQLite3.Step to determine what happened during that step. The resulting error code may be SQLite3.BUSY, SQLite3.ROW, SQLite3.DONE, SQLite3.ERROR, or SQLite3.MISUSE.

Whenever another row of result data is available, the error code will be SQLite3.ROW.

When all processing is complete, the error code will either be SQLite3.DONE or SQLite3.ERROR. SQLite3.DONE indicates that the statement completed successfully and SQLite3.ERROR indicates that there was a run-time error. It is incorrect to attempt calling SQLite3.Step again after it has generated SQLite3.DONE or SQLite3.ERROR.

dbHandle:

(database) The handle to the database that you want to operate on.

Note: The database must have a compiled query in it or a syntax error will be generated.

Returns:

Nothing.

Example:

sql="select * from Users";
SQLite3.Compile(db,sql);
err = Application.GetLastError();
if (err ~= SQLite3.OK ) then
    Debug.Print("Error (".. err .."): ".. SQLite3.GetErrorString(err) .. "\r\n");
else
    Debug.ShowWindow(true);

    repeat
        SQLite3.Step(db);
	    err = Application.GetLastError();
        if (err == SQLite3.ROW) then
            tbRow=SQLite3.GetRowDataAssoc(db)
            for strColumnName,strValue in pairs(tbRow) do
                Debug.Print(strColumnName.." = "..strValue.."\r\n");
            end
            Debug.Print("----\r\n");
        end
    until (err ~= SQLite3.ROW)

    SQLite3.Finalize(db)
    err = Application.GetLastError();
    if (err ~= SQLite3.OK ) then
        Debug.Print("Error (".. err .."): ".. SQLite3.GetErrorString(err) .. "\r\n");
    end
end
