Project.Associate


Project.Associate ( 

(string) Extension = "abcd",

(string) Exe = "_ExeFolder.."\\"..Project.GetDetails().ExeName",

(string) Icon = "_ExeFolder.."\\"..Project.GetDetails().ExeName",

(string) Short Description = "Project",

(string) Long Description = "Project Manager" )

Description

Creates a file and icon association with the program for a specified file extension.

Note: Warning: If you are compiling your project with "MoleBox Virtualization Solution" then be sure to have your associated file icons OUTSIDE of the compiled exe and be sure that the association icon paths point to them as well or Windows will not be able to see them.

Parameters

Extension

(string) The file extension for which to create the association.

Note: You may include OR exclude the "." before the file extension letters.

Exe

(string) The full path to the exe file which will be executed when the specified file type is clicked.

Note: Leave this blank to use the default program exe file.

Icon

(string) The full path to the icon file that will be used for this file type.

Note: Leave this blank to use the default program icon file.

Short Description

(string) It's best to use the project name or something as simple for this.

Long Description

(string) This text will show up when a user mouse-overs a file.

Returns

This action dose not return any value.

Project.GetFileCode


(table) Project.GetFileCode ( 

(string) Extension = ".txt" )

Description

Dermines whether or not the program was opened by a calling file with the specified extension.

Parameters

Extension

(string) The allowed file type.

Returns

(table) Returns a table indexed by Code and File.

If the program was called by the specified file type then the Code value is 1 and the File value is the path to the calling file.

If the program was called by a file but not of the specified type then the Code value is -1 and the File value is the path to the calling file.

If the program was not called by a file then the Code value is 0 and the File value is a blank string.


Example

local tReturn = Project.GetFileCode(".skde");

if tReturn.Code == 1 then
Dialog.Message("Success", "Program opened with file...\r\n"..tReturn.File);
elseif tReturn.Code == -1 then
Dialog.Message("Error", "That file type is not supported.");
end

Project.ProcessVersion


(string) Project.ProcessVersion ( 

(string) Version File = "",

(number) Version Segments = 4,

(number) Max Number = 9 )

Description

Updates and/or gets the project's internal version (not the file version).

Note: Every time the project runs when it is uncompiled, the version is incremented and returned, If the project is compiled the verion is just returned.

Parameters

Version File

(string) The full path to the version file. Leave this blank to use the default file.

Version Segments

(number) The number of version points to use (3 would look like 1.1.3, while 5 would look like 2.6.2.1.5)

Max Number

(number) The highest number to which each version segment will ascend.

Note: To use a 0-9 system set the Max Number to 9.

Returns

(string) Returns a string containing the verion number of the program (but not the file version).

(boolean) Project.UpdateRuntime ( 

(string) Runtime = "D:\Downloads\MyFile.exe",

(string) CDD = "D:\Downloads\MyFile.cdd",

(boolean) Show Update Window = true,

(string) Background Color = 0,

Title

(string) The title of the command window.

Note: Leave this blank to use the default value.

Pre-timeout Message

(string) The message shown right before the update starts and before the timeout.

Note: Leave this blank to use the default value.

Post-timeout Message

(string) The message shown right before the update starts and after the timeout.

Note: Leave this blank to use the default value.

Restart Runtime

(boolean) Whether or not to restart the runtime after the update is complete.

Force Shutdown

(boolean) By default, the runtime will be updated when the application shuts down. You can set this argument to true to force the application to shutdown.

Note: If there is an error, such as missing update files or invalid arguments passed to the function, the application will not be shutdown.

Returns

(boolean)
Returns true if all of the arguments were passed correctly and if the execution point is valid and false if not.

Returns nothing if a forced shutdown is requested.

Valid Execution Points:
For the Runtime: "Hard Drive","Removable Drive","Network Drive","Temp Folder"
For the cdd file: "Hard Drive","Removable Drive","Network Drive"
(You can get the execution point by calling the Project.GetExecutionPoint() function.)

Note: This function does not check for the existence of the update files until the shutdown procedure.
During this stage it will still return true even if the file paths are invalid.
At shutdown, however, when the runtime update begins, it will be aborted if the file paths are invalid.
